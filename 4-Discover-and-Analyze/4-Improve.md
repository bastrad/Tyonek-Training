# 4 · Improve

**DMAIC — Improve.** Turn the prioritized analysis into recommended actions and align leadership.
This is where discovery becomes a plan — but the *sustained* execution happens in the standing
program (see Control).

## Deliverable — the Initial Findings Report
One decision-ready report leadership can act on:
1. **Current-state summary** — RAG picture by subsidiary (how many 🟢/🟡/🔴).
2. **Top risks** — the vital few, in plain compliance/contract terms.
3. **Quick wins** — what we'll fix immediately, at low cost.
4. **Recommended path** — how we move into verification → gap analysis → the improvement loop.
5. **Resourcing** — what's needed (time, tools, ISD support) to close systemic gaps.

## Route each gap to an action
| Finding | Route | Where it goes |
| --- | --- | --- |
| Training exists, just needs delivery | **Train** | Standing loop |
| No material exists (🔴) | **Develop** (ISD/ADDIE) | Build new courseware |
| Training happened, no proof | **Records** | Fix documentation/tracking |
| Requirement unclear | **Verify** | [Requirements Verification](../3-Requirements/Requirements-Verification-Process.md) |

## Pilot before scaling
Lean Six Sigma improves in small, proven steps. Pick one subsidiary or one high-risk process,
run the fix (e.g., stand up cert tracking + close the top Red gaps), confirm it works, then
standardize and scale — rather than boiling the ocean.

## Connect to the standing program
The "Develop / Train / Assess / Document" work is already defined — Improve hands off to it:
- New material → [Continuous-Improvement Loop, Stage 1 Develop](../6-Sustain-and-Improve/Continuous-Improvement-Loop.md)
- Gap tracking → [Gap Analysis template](templates/Training-Needs-Gap-Analysis.md)

## Improve-phase checklist
- ▢ Initial Findings Report drafted and reviewed with leadership
- ▢ Quick wins executed
- ▢ Systemic gaps scoped as projects with owners/dates
- ▢ Pilot chosen and underway

**Output:** an agreed action plan and the first improvements in motion.
