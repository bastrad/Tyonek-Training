# Phase 4 · Discover & Analyze

*Learn what training the company **already has**, then compare it to the requirements (Phase 3)
to find the **gaps**. Structured as a Lean Six Sigma **DMAIC** project.*

## The DMAIC sequence (work through in order)
| # | Step | What it does |
| --- | --- | --- |
| 1 | [Define](1-Define.md) | Charter the discovery: scope, stakeholders, what "good" means (SIPOC, CTQ) |
| 2 | [Measure](2-Measure.md) | Collect current-state data — records, materials, systems (Gemba, no judging yet) |
| 3 | [Analyze](3-Analyze.md) | Rate each job (RAG), find gaps and root causes, prioritize the vital few |
| 4 | [Improve](4-Improve.md) | Recommendations + the Initial Findings Report |
| 5 | [Control](5-Control.md) | Hand off to the standing program (Phases 5–6) |

## The gap analysis
- [templates/Training-Needs-Gap-Analysis.md](templates/Training-Needs-Gap-Analysis.md) — the
  fillable worksheet: required (from Phase 3) vs. current, and how each gap is routed.
- [examples/](examples) — a **worked Welding example** (filled Requirements Register + Gap

## Where gaps go next
Every gap routes to an action in the next phases:
- **Train** or **Develop** new material → [Phase 5](../5-Develop-Deliver-Assess/)
- **Records / audit** fixes → [Phase 6](../6-Sustain-and-Improve/)
