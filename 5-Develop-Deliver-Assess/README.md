# Phase 5 · Develop · Deliver · Assess

*Close the gaps found in Phase 4: build any training that's missing, deliver it, and verify that
people are actually competent — not just that they attended.*

## The three steps
1. **Develop** (only where a gap needs new material) — build the training using **ISD / ADDIE**
   (Analyze → Design → Develop → Implement → Evaluate). Tyonek Mission Support's Instructional
   Systems Design team is an in-house courseware capability. The training document lives in the
   job's own folder (Phase 2, its `README.md`) — that *is* the standard.
2. **Deliver (Train)** — deliver the standardized training. Blend **on-the-job learning** with
   related instruction, mirroring the apprenticeship structure in each job's `TRADE-PROGRESSION.md`.
3. **Assess** — verify competency with a **knowledge check + practical demonstration**, signed
   off by a qualified evaluator. Pass → record it (Phase 6). Fail → retrain.

## Lean note
This is the **Jidoka** (build-quality-in) point of the program: competency is the gate, and
unqualified work doesn't proceed. Develop → Deliver → Assess is the "Do" of the continuous loop
described in [Phase 6](../6-Sustain-and-Improve/).
