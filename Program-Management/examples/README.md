# Worked Examples

Filled-in examples that show what the Program-Management templates look like when
completed. **These are demonstrations** — start from the blank templates in
[`../templates/`](../templates) for real work.

| Example | Shows |
| --- | --- |
| [Welding — Requirements Verification Register](Welding-Requirements-Verification-Register.md) | The **"required" side**: every training/qualification Welding must have, with its authoritative source, who it applies to, and the evidence needed. |
| [Welding — Training Needs / Gap Analysis](Welding-Training-Needs-Gap-Analysis.md) | The **comparison**: required vs. current, and how gaps get found and routed. Includes a real "PENDING" version and an illustrative completed version. |

**Process this example belongs to:**
`Tyonek-Manufacturing-Group/Value-Stream/03-Welding/`

> **Important honesty note:** the *required* side is built from public governing material
> (OSHA, AS9100, NADCAP, AWS) plus Tyonek's public capability statements. The *current-state*
> data shown in the illustrative gap analysis is **fictional**, invented only to demonstrate
> how a finished analysis reads. Real current-state data must be collected from the company.
